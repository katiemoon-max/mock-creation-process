# Exam Diagram Generator

Generate publication-ready diagrams for exam questions using matplotlib. All diagrams use a black/grey/white palette for accessibility. Both blank (question paper) and completed (model answer) versions are generated by default.

## Input

The user provides:
- **Diagram description** (required): What the diagram should show, including axes, labels, data values, and key features
- **Question number** (optional): e.g. "Q5" — used for file naming. Infer from context if not given
- **Figure number** (optional): e.g. "Fig. 5.1" — used for the title on the diagram. Infer from context if not given
- **Versions needed** (optional, default "both"): "blank", "answer", or "both"
- **Style overrides** (optional): font size, figure dimensions, DPI

If the description is ambiguous, ask clarifying questions before generating:
- What are the axes? What units?
- What data should be plotted?
- What labels and annotations are needed?
- What must the student do with this diagram (read values, plot data, complete it, interpret it)?

## Process

### Step -1: Check dependencies

Before generating any diagram, verify that Python, matplotlib, and numpy are available:

```bash
python -c "import matplotlib; import numpy; print(f'matplotlib {matplotlib.__version__}, numpy {numpy.__version__}')"
```

If this fails:
- If Python is not installed, inform the user and suggest installing it (do not install automatically)
- If matplotlib or numpy are missing, install them: `pip install matplotlib numpy`
- Re-run the check to confirm installation succeeded before proceeding

### Step 0: Determine output location

- Look for an existing `diagrams/` directory in the current project. If found, use it. If not, create one
- Confirm the question number and figure number with the user if not provided or inferable

### Step 1: Classify the diagram type

Determine which category from the Diagram Type Reference (below) the request falls into. If the diagram is in Tier 3 (outside matplotlib's strengths), warn the user and suggest alternatives before proceeding.

### Step 2: Determine blank vs answer versions

Apply this decision tree:

1. **Stimulus diagram** (students read from it — e.g. a pre-plotted graph, a black-body curve, an apparatus setup): generate **one version only**
2. **Completion diagram** (students must draw/plot/complete something — e.g. draw force arrows, plot data, complete a ray path): generate **both versions**
   - Blank: the setup/context only (axes, apparatus, partial diagram)
   - Answer: the completed version with all student-drawn elements added
3. **Answer-only diagram** (only relevant to the model answer — e.g. a correctly plotted graph not shown in the question): generate **one version (answer only)**

State which versions you are generating and why.

### Step 3: Write the Python script

Generate a single `.py` file following the Script Template below. Key rules:
- One function per diagram variant (e.g. `q5_refraction_blank()`, `q5_refraction_answer()`)
- All functions self-contained: create figure, plot, save, close
- `__main__` block calls all functions
- Use the house style `rcParams` block at the top

### Step 4: Run the script

Execute the Python script via Bash. Verify all expected PNG files were generated. Display the images to the user for review.

### Step 5: Generate alt text

For each PNG, write descriptive alt text:
- Single descriptive sentence (can be long)
- Describe spatial layout, key labels, data values, and relationships
- Written for a screen reader user who cannot see the image
- No interpretation or explanation — just description
- Include specific numerical values where relevant

Present in ready-to-paste format:
```
![A speed-time graph showing a trapezium shape: speed increases linearly from 0 to 12 m/s over the first 8 seconds, remains constant at 12 m/s until 28 seconds, then decreases linearly to 0 at 32 seconds. Major gridlines every 4 seconds and 2 m/s; minor gridlines every 1 second and 0.5 m/s.](diagrams/Q01_speed_time_graph.png)
```

### Step 6: Quality checks

Before presenting diagrams, verify:
1. **Palette compliance**: Only black, grey shades (#555555, #999999, #CCCCCC, #E0E0E0), and white used. No colour
2. **Label legibility**: Font sizes appropriate for the figure size. No overlapping labels
3. **Axis format**: Labels use `quantity / unit` format (e.g. `time / s`, `force / N`). Unit notation matches the exam board convention — negative exponents for A Level (`m s$^{-1}$`), fraction for GCSE (`m/s`)
4. **Figure title**: Matches the expected numbering scheme (`Fig. N.M` italic, right-aligned)
5. **Blank/answer separation**: Blank version contains NO answer elements
6. **Gridline precision**: If students must read values, gridlines allow reading to the required precision (typically ±½ a small square)
7. **Scientific accuracy**: Data shapes are physically realistic (exponentials approach zero asymptotically, not abruptly; oscillations decay correctly)
8. **Tight layout**: `bbox_inches="tight"` used to avoid excessive whitespace
9. **White background**: `facecolor="white"` explicitly set (prevents transparent backgrounds)
10. **File naming**: Follows the naming convention below

## Script Template

```python
"""Generate diagrams for [context description]."""

import numpy as np
import matplotlib.pyplot as plt
import matplotlib.patches as patches
from pathlib import Path

OUT = Path(__file__).parent
DPI = 300

# ── House style ──────────────────────────────────────────────────────────────
plt.rcParams.update({
    "font.family": "serif",
    "font.size": 11,
    "axes.linewidth": 1.0,
    "axes.edgecolor": "black",
    "xtick.direction": "in",
    "ytick.direction": "in",
    "xtick.major.width": 1.0,
    "ytick.major.width": 1.0,
    "xtick.minor.width": 0.6,
    "ytick.minor.width": 0.6,
    "figure.facecolor": "white",
    "axes.facecolor": "white",
    "grid.color": "#CCCCCC",
    "grid.linewidth": 0.5,
})


# ═══════════════════════════════════════════════════════════════════════════════
# Q[N] Fig. [X] — [description] (BLANK)
# ═══════════════════════════════════════════════════════════════════════════════
def q[n]_[short_name]_blank():
    fig, ax = plt.subplots(figsize=(8, 5))

    # ... plotting code ...

    ax.set_xlabel("quantity / unit", fontsize=12)
    ax.set_ylabel("quantity / unit", fontsize=12)
    ax.set_title("Fig. N.M", fontsize=12, fontstyle="italic", loc="right")

    fig.tight_layout()
    fig.savefig(OUT / "Q0N_short_name_blank.png", dpi=DPI, bbox_inches="tight", facecolor="white")
    plt.close(fig)
    print("  Q[N] Fig. [X] (blank) saved")


# ═══════════════════════════════════════════════════════════════════════════════
# Q[N] Fig. [X] — [description] (ANSWER)
# ═══════════════════════════════════════════════════════════════════════════════
def q[n]_[short_name]_answer():
    fig, ax = plt.subplots(figsize=(8, 5))

    # ... plotting code (includes answer elements) ...

    fig.tight_layout()
    fig.savefig(OUT / "Q0N_short_name_answer.png", dpi=DPI, bbox_inches="tight", facecolor="white")
    plt.close(fig)
    print("  Q[N] Fig. [X] (answer) saved")


if __name__ == "__main__":
    q[n]_[short_name]_blank()
    q[n]_[short_name]_answer()
    print("\nAll diagrams generated successfully.")
```

## Styling Defaults

| Parameter | Default | How to override |
|-----------|---------|-----------------|
| DPI | 300 | "use 200 DPI" |
| Font size | 11 pt | "use 14pt font" |
| Font family | serif | "use sans-serif" |
| Figure size | type-dependent (see below) | "make it 10x6 inches" |
| Colour palette | black/grey/white only | **Not overridable** — exam accessibility rule |
| Tick direction | inward | "use outward ticks" |
| Grid style | #CCCCCC, 0.5pt | "use heavier gridlines" |

### Default figure sizes by type

| Diagram type | Default size |
|-------------|-------------|
| Standard graph | (8, 5) |
| Graph with gridlines for reading values | (8, 6) or (10, 6) for wide ranges |
| Ray/optics diagram | (10, 5) |
| Circuit diagram | (8, 6) |
| Force/vector diagram | (6, 6) |
| Apparatus schematic | varies by content |
| MCQ option set | (6, 6) |

### Greyscale palette reference

| Shade | Hex | Use for |
|-------|-----|---------|
| Pure black | `"black"` | Primary lines, text, arrows, data points |
| Dark grey | `"#555555"` | Secondary labels, dimension lines |
| Medium grey | `"#999999"` | Dashed construction lines, normals |
| Light grey | `"#CCCCCC"` | Major gridlines |
| Very light grey | `"#E0E0E0"` | Minor gridlines, fill regions (glass blocks, shaded areas) |
| White | `"white"` | Background, unfilled regions |

## File Naming Convention

**Format**: `Q[NN]_[short_description]_[version].png`

- Question number: zero-padded to 2 digits (Q01, Q05, Q25)
- Short description: snake_case, concise
- Version suffix: `_blank` or `_answer` (omit if only one version)

Examples:
- `Q01_speed_time_graph.png` — single-version stimulus graph
- `Q05_semicircular_block_blank.png` / `Q05_semicircular_block_answer.png` — paired versions
- `Q03_Fig3_force_diagram_blank.png` — with figure reference in name

## Diagram Type Reference

### Tier 1 — Excellent matplotlib fit

| Type | Key features | Typical use |
|------|-------------|-------------|
| Line graphs (v-t, s-t, F-x, p-V, I-V) | `ax.plot()`, `ax.grid()`, tick control | Most common. Use minor gridlines for precision reading |
| Bar charts | `ax.bar()` | Biology, geography, data analysis |
| Scatter plots | `ax.scatter()` | Often with best-fit line overlay |
| Log-log and semi-log plots | `ax.set_xscale("log")` | Error bars equal in log space |
| Error bar plots | `ax.errorbar()` | Asymmetric error bars supported |
| Histograms | `ax.hist()` | Count distributions, decay data |
| Blank graph paper / axes | Grid only, no data | "Plot these points" questions |
| Decay/growth curves | `np.exp()` | Half-life markers as dashed lines |

### Tier 2 — Good fit (requires patches/annotations)

| Type | Key features | Notes |
|------|-------------|-------|
| Ray diagrams (optics) | `ax.plot()`, `ax.annotate()` with arrowprops, `patches.Arc` | Arrowheads on rays. Lenses as double-headed arrows or thin shapes |
| Force/vector diagrams | `ax.annotate()` with `arrowstyle="-\|>"` | Scale arrow lengths proportionally to magnitude |
| Wave diagrams | `np.sin()` with `ax.plot()` | Label amplitude, wavelength, phase |
| Simple circuit diagrams | `patches.Rectangle`, lines, text labels | Adequate for simple series/parallel circuits |
| Apparatus schematics | `patches.Rectangle`, `patches.Circle`, lines | Rulers, blocks, springs, lenses, slits |
| Field line patterns | `ax.streamplot()` or manual curves | Simple dipole patterns; manual for custom |
| Coordinate geometry | `ax.plot()`, `patches.Polygon` | Triangles, circles, tangent lines |
| Instrument readings (CRO, multimeter) | Rectangles with internal lines and text | Display screens, dial readings |
| Sankey diagrams (simple) | `patches.FancyArrowPatch` | Energy transfer diagrams |

### Tier 3 — Warn the user

| Type | Issue | Suggest instead |
|------|-------|----------------|
| Complex 3D diagrams | Matplotlib 3D is limited | Hand-drawn or specialist 3D tool |
| Detailed anatomical diagrams | Very labour-intensive with patches | Stock images or illustration software |
| Complex chemical structures | Tedious bond-by-bond drawing | RDKit or ChemDraw |
| Detailed apparatus (many components) | Possible but very time-consuming | Simplified schematic, or hand-drawn |
| Maps or geographical features | Poor cartographic support | Dedicated mapping tools |
| Photorealistic images | Not possible | Photography or illustration |

If a request falls into Tier 3, warn the user:
> "This diagram type is outside matplotlib's strengths. I can create a simplified schematic version, or you may prefer to use [alternative tool]. Would you like me to proceed with a simplified version?"

## Common Patterns

### Graph with gridlines for reading values

```python
# Major gridlines for labelled values
ax.set_xticks(np.arange(0, x_max + 1, major_step))
ax.set_yticks(np.arange(0, y_max + 1, major_step))

# Minor gridlines for precision reading
ax.set_xticks(np.arange(0, x_max + 1, minor_step), minor=True)
ax.set_yticks(np.arange(0, y_max + 1, minor_step), minor=True)

ax.grid(True, which="minor", linewidth=0.15, color="#E0E0E0", zorder=0)
ax.grid(True, which="major", linewidth=0.5, color="#AAAAAA", zorder=0)
```

### Arrow annotations (force/ray diagrams)

```python
ax.annotate("", xy=(x_end, y_end), xytext=(x_start, y_start),
            arrowprops=dict(arrowstyle="-|>", color="black", lw=1.5))
ax.text(x_label, y_label, "F", fontsize=12, ha="center", va="center")
```

### Dashed construction lines (normals, extensions)

```python
ax.plot([x1, x2], [y1, y2], "k--", linewidth=0.8, color="#999999")
```

### Shaded regions (glass blocks, objects)

```python
polygon = patches.Polygon(vertices, closed=True, facecolor="#E0E0E0",
                          edgecolor="black", linewidth=1.2)
ax.add_patch(polygon)
```

### Angle arcs with labels

```python
arc = patches.Arc((cx, cy), width, height, angle=0,
                  theta1=start_deg, theta2=end_deg, color="black", lw=1.0)
ax.add_patch(arc)
ax.text(label_x, label_y, r"$\theta$", fontsize=11, ha="center")
```

## Limitations

matplotlib is a general-purpose plotting library, not a diagram editor. Keep these limits in mind:

- **Text layout**: matplotlib has limited control over text flow. Long labels may overlap — adjust positions manually
- **Curved arrows**: Supported via `FancyArrowPatch` but fiddly to position precisely
- **3D**: `mpl_toolkits.mplot3d` exists but produces low-quality results compared to dedicated 3D tools
- **Symbols**: Standard physics symbols (resistor zigzag, capacitor plates, etc.) must be drawn manually from primitives — there is no built-in component library
- **Interactive elements**: matplotlib produces static images only
- **Complex curves**: Bezier curves are possible but require manual control point placement
